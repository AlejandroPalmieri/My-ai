# AgentOS Personal v0.3.0 Release Notes

v0.3.0 is a release checkpoint for the local AgentOS runtime. It keeps the
project local-first while adding streaming chat, provider adapters, explicit
retrieval, safe agent tool-calling, stronger evals, and a documented MCP SDK
decision.

## Summary

- Streaming chat works for one-shot and interactive chat when the active provider
  supports it.
- Provider adapters cover `local-stub`, OpenAI, OpenAI-compatible, OpenRouter,
  Anthropic, and Ollama behind one model client boundary.
- Retrieval from technical memory and Strategic Brain is explicit opt-in only.
- Agent runs can call only allowlisted internal tools with bounded steps and
  policy checks.
- Evals cover providers, streaming, context, retrieval, agents, tools, and safety.
- Formal Python MCP SDK adoption is deferred; the current MCP server remains a
  local JSON-RPC STDIO implementation.

## Security Posture

- `.agentos/`, `.env`, runtime databases, traces, backups, logs, and exports are
  ignored by git.
- API key values are not stored in model configuration; only environment variable
  names are stored.
- No unrestricted shell execution tool exists in the built-in tool registry or MCP
  tool surface.
- Retrieval is off by default and must be enabled per request or per interactive
  session.
- Usage databases, traces, backups, and other runtime-state files remain local
  `.agentos/` artifacts and should not be committed.

## Known Limitations

- MCP uses the custom local JSON-RPC STDIO implementation; formal Python MCP SDK
  adoption is deferred.
- Provider adapters use `httpx` and local config, not vendor SDKs.
- Strategic Brain does not yet implement embeddings, graph reasoning, PDF
  ingestion, or LLM synthesis.
- Agent runs are bounded local tool loops, not autonomous background workers.
- Backup integrity checks and encryption are not implemented yet.

## v0.4.0 Milestones
