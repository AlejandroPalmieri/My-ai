# v0.3.0 Implementation Plan

v0.3.0 strengthens the local runtime with deterministic eval coverage for model
provider metadata, streaming, retrieval, context usage, bounded agent runs, safe
tool-calling, and safety policy checks.

## Implemented in this slice

- Added categorized eval cases under `src/agentos/evals/cases.py`.
- Added reusable assertion helpers under `src/agentos/evals/assertions.py`.
- Extended the eval runner to filter by category and capture failures instead of
  aborting the whole run.
- Added JSON and Markdown report writing plus lookup by latest report or report id.
- Added CLI support for `agentos eval run --category ...` and
  `agentos eval report ...`.

## Local-only constraints

- Provider evals use `local-stub` or metadata validation only.
- Missing API key checks validate clear warnings without network calls.
- Eval workspaces live under `.agentos/evals/workspace/`.
- Safety evals prove policy behavior without reading `.env` contents.
- Tool and agent evals use allowlisted internal tools only; no shell execution is
  exposed.

## Verification targets

```powershell
ruff check .
pytest
agentos eval run --category safety --root .
agentos eval run --category providers --root .
```
